📌 Raspberry Pi 環境設定

  • 樹莓派為基於 Linux 的單板電腦，本課程使用 Raspberry Pi 3 Model B
  • 開機需要 SD 卡（可使用 Raspberry Pi Imager 建立）
  • 預設帳密：pi / raspberry
  • 可透過 MobaXterm 使用 SSH 遠端連線；VNC Viewer 可圖形介面操作

🟢 GPIO 與 LED / Button 控制

  • GPIO：General Purpose Input/Output
  • GPIO 可設為 Input / Output / Alternate Function
  • 控制 LED 需搭配 電阻與杜邦線 接在麵包板上
  • GPIO.setmode(GPIO.BOARD) 表示以實體腳位編號為準
  • PWM（Pulse Width Modulation）：脈波寬度調變，用於產生類比效果（例如呼吸燈）

💡 電壓與電流

  • 電壓（Volt, V）：電位差
  • 電流（Ampere, A）：電子流動量
  • 電阻（Ohm, Ω）：阻礙電流流動的能力
    👉 歐姆定律：V = I × R

🔊 蜂鳴器

  • 分為 有源（Active）與 無源（Passive）蜂鳴器
    • 有源：通電即響，內建震盪器
    • 無源：須提供 PWM 方波才能響
  • 音調控制：改變頻率與週期可產生不同聲音
  • 音樂播放：可依照音階頻率改變來演奏旋律

🌈 光敏電阻 / 雨滴感測器 / 土壤濕度

  • 光敏電阻：光越強，電阻越小，電壓變化可做為輸出
  • 雨滴感測器：無水 → 高電位，有水 → 低電位
  • 土壤濕度感測器：濕 → 導電性強 → 電阻小，乾 → 電阻大

📏 超聲波感測器（HC-SR04）

  • 可量測距離範圍：約 2cm ~ 400cm
  • 原理：發送超聲波 → 偵測回波時間 → 根據聲速計算距離
    • 距離(cm) = 時間 × 聲速 / 2
    • 聲速 = 331 + 0.6 × 溫度

🧱 Thread vs Process (執行緒與程序)

Thread (線程)

  • 較輕量，屬於 process 的一部分，共享記憶體與資源
  • 多執行緒需使用 Lock 鎖定 避免資源競爭
  • 低 context switching 成本

Process (程序)

  • 較重，記憶體獨立，需額外機制（如 shared memory）溝通
  • 適合多核心運算，缺點是溝通成本高、資源需求大

Python 限制：GIL（Global Interpreter Lock）

  • 單執行緒限制，Python 在多執行緒下效能受限

📺 SPI 傳輸協定與 TFT LCD

  • SPI（Serial Peripheral Interface）：全雙工、高速、主從式傳輸
  • 常見腳位：MOSI、MISO、SCLK、CS
  • SPI 比 I²C 更高速但需更多線路

LCD 模組（ILI9341）

  • 使用 4線式 SPI
  • 支援繪圖、顯示圖片與文字（搭配 Pillow 套件）
  • 可搭配觸控功能（需額外驅動與接線設定）

📡 I²C 傳輸與感測器整合

  • I²C 為兩線式通訊：SDA（資料）、SCL（時脈）
  • Master-slave 結構，每個裝置需有唯一地址
  • 優點：支援多設備、線路簡單；缺點：速度較慢
  • 常見應用：LM75A 溫度感測器、MPU-6050（加速度 + 陀螺儀）

🧠 小提醒

  • 每種感測器通常會有一個 D0（數位） 與 A0（類比） 輸出腳位（如有支援）
  • 雖然不考程式碼，但了解感測器如何回傳訊號（高電位 / 低電位）非常關鍵
  • 考試以 選擇題為主，可多練習判斷感測器行為與輸出特性
